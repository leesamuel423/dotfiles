TL;DR
Claude Code hooks (PreToolUse, PostToolUse, Notification) let you attach shell scripts that fire before/after every tool call. One dev used three hooks to auto-format code, block .env edits, and pipe alerts to desktop notifications — eliminating ~20 manual Prettier runs/day and removing the need to babysit the terminal.The Setup
Every time Claude edits a file, you're manually running formatters afterward. Every time it finishes a long task, you're alt-tabbing to check. And there's always the risk it touches a .env or secrets file. These are small paper cuts that compound across a full day of agentic coding.How It Works
Hooks live in .claude/settings.json (project) or ~/.claude/settings.json (global). Three hooks cover most needs:
PostToolUse — auto-runs Prettier after any Edit or Write. Matcher: Edit|Write, command pipes jq -r '.tool_input.file_path' into xargs npx prettier --write.
PreToolUse — blocks .env edits deterministically. Reads the file path from stdin JSON; if it matches .env, exits with code 2 (which blocks the tool call).
Notification — pipes Claude alerts to macOS osascript for native desktop notifications.
The key insight: hooks receive tool input as JSON on stdin, so you can inspect what Claude is about to do and gate it programmatically.
Real-World Results
Eliminated ~20 manual npx prettier --write runs per day
.env files are now deterministically untouchable — no more "oops" moments
Desktop notifications cut context-switching; author reports working on other tasks while Claude runs in background
Setup time: ~3 minutes per hook
How to Try This
Add the PostToolUse formatter hook to .claude/settings.json — swap Prettier for your formatter (gofmt, rustfmt, etc.)
Add the PreToolUse security hook globally in ~/.claude/settings.json to block sensitive paths
Run a normal Claude Code session and watch hooks fire automatically in the tool output
Limitations and Gotchas
Hooks are shell commands — complex logic should live in external scripts, not inline JSON
exit 2 blocks the tool call silently; make sure you log why or Claude may retry confusedly
Notification hook is macOS-specific (osascript); Linux needs notify-send equivalent

Hooks are shell commands that fire at specific lifecycle points (PreToolUse, PostToolUse, PostResponse, etc.). Add them to .claude/settings.json at project or global level. The killer combo:

Exit code 0 = proceed, 2 = block action. That's the whole API.

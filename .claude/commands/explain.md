# Explain Code

Provide a clear, educational explanation of how the specified code works.

## Instructions

Given a file, function, module, or code concept:

1. **Read the target code** and its imports/dependencies
2. **Explain at multiple levels**:
   - **High level**: what it does and why it exists (1-2 sentences)
   - **Flow**: step-by-step walkthrough of the logic
   - **Key details**: important patterns, algorithms, or non-obvious behavior
3. **Note connections** to other parts of the codebase (callers, dependents)
4. **Highlight gotchas** — edge cases, assumptions, or subtle behavior that could trip someone up

Keep explanations clear and practical. Use the code itself as reference points.
